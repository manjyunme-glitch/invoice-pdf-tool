# 发票 PDF 筛选与整理工具 v5.1

一个面向 Windows 的发票处理工具，帮助你把 Excel 里的发票清单和本地 PDF 发票文件对应起来。

它既可以按文件名规则整理一批零散 PDF，也可以从复杂的 Excel 工作簿里筛选出需要的发票号，再批量导出对应 PDF。适合进项发票、抵扣发票、税局导出表、内部台账和对账表等日常财务场景。

## 为什么做这个工具

很多发票筛选工作都卡在这几件事上：

- Excel 很复杂，一个工作簿里有很多个 Sheet
- 不同 Sheet 的列名不统一，既有税局导出的表，也有自己维护的表
- 发票号、公司名、是否抵扣等信息散落在不同列里
- PDF 数量很多，人工逐个查找和复制很慢
- 每个月都要重复同样的动作

这个工具的目标，就是把这套重复劳动尽量自动化。

## 适用场景

- 月度进项发票筛选
- 抵扣发票筛选
- 税局导出清单对应 PDF 回收
- 按公司归档 PDF 发票
- 从复杂台账里按条件筛出需要的发票资料

## 功能亮点

### 1. 发票筛选

- 打开 `.xlsx` / `.xls` 文件
- 自动识别工作簿中的多个 Sheet
- 手动切换目标 Sheet
- 从 Excel 中提取发票号
- 到本地 PDF 目录中匹配对应文件
- 先预览，再正式导出
- 生成筛选结果报告

### 2. 多 Sheet 工作簿分析

- 显示所有 Sheet 的名称、行列规模和状态
- 自动识别候选发票列
- 自动识别候选公司列
- 标记推荐 Sheet、可用 Sheet、异常 Sheet
- 提供样本预览，便于人工确认

这部分是 `v5` 系列的重点改进，适合复杂 Excel。

### 3. 行级条件筛选

除了按发票号筛选，还可以根据其他列继续缩小范围，例如：

- 只筛选“是否抵扣 = 是”的发票
- 排除某些公司名称、临时标记、脏数据
- 按某一列做包含、不包含、等于、不等于筛选

支持的模式：

- `不过滤`
- `等于任一`
- `包含任一`
- `不等于任一`
- `不包含任一`

### 4. 自动导出目录

支持按“Excel 所在目录 + 当前 Sheet 名称”自动创建导出文件夹。

例如当前 Excel 在：

```text
F:\工作文档\2025年8月
```

当前 Sheet 是：

```text
8月进项
```

那么导出目录会自动变成：

```text
F:\工作文档\2025年8月\8月进项
```

这对“进项”和“抵扣”这种按 Sheet 分开归档的流程非常实用。

### 5. 发票整理

- 扫描文件夹中的 PDF
- 按文件名规则解析公司名
- 自动移动到对应公司目录
- 支持递归扫描
- 支持预览、勾选、撤销

### 6. 历史记录与回滚

- 查看整理历史
- 查看筛选历史
- 关键字搜索
- 回滚导出结果
- 回滚关联报告

### 7. 界面体验

- 白天 / 黑夜双主题
- 默认窗口更紧凑
- 结果区表格化展示
- 设置页和筛选页都支持滚动
- 切换 Sheet 时自动重置行筛选条件，避免误带上一个 Sheet 的条件

## 快速开始

### 方式一：直接使用 EXE

1. 从 GitHub `Releases` 页面下载 `invoice-pdf-tool-v5.1-windows-x64.exe`
2. 双击打开
3. 进入“发票筛选”页
4. 选择 Excel 文件和 PDF 文件夹
5. 点击“分析工作簿”
6. 选择要筛选的 Sheet
7. 如有需要，设置条件列、条件值和排除公司
8. 点击“预览匹配”
9. 确认无误后点击“开始筛选并导出”

### 方式二：从源码运行

```bash
pip install -r requirements.txt
python 发票处理工具v5.py
```

## 典型使用流程

### 进项发票筛选

1. 打开 Excel 文件
2. 选择 `8月进项` 这类 Sheet
3. 检查系统识别到的发票列和公司列
4. 如有需要，设置排除公司
5. 开启“按 Excel 同目录 + 当前工作表自动建导出文件夹”
6. 先预览，再导出

### 抵扣发票筛选

1. 打开 Excel 文件
2. 选择 `8月抵扣` 这类 Sheet
3. 将条件列设为“是否抵扣”
4. 选择筛选模式“等于任一”
5. 条件值填“是”
6. 预览匹配结果
7. 导出匹配到的 PDF

### PDF 归档整理

1. 进入“发票整理”
2. 选择发票 PDF 所在目录
3. 根据命名规则检查公司名段位
4. 扫描文件
5. 预览后执行整理
6. 如有误可撤销

## PDF 文件命名规则

工具默认通过文件名分段来提取关键信息。

常见格式示例：

```text
dzfp_发票号码_公司名称_日期.pdf
```

默认思路是：

- 使用 `_` 作为分隔符
- 在设置里配置“公司名称所在段”和“发票号码所在段”

如果你的 PDF 文件名格式不同，也可以在设置页调整规则预设或手动修改段位。

## Excel 支持说明

工具支持从复杂工作簿中做筛选，但建议注意下面几点：

- 不同 Sheet 的列名可以不一致
- 工具会尝试自动识别“发票列”和“公司列”
- 如果自动识别不准确，可以手动指定列
- 如果 Excel 已经把超长数字按数值存储并丢失精度，任何工具都无法完全还原原始发票号

也就是说：

- 工具已经尽量避免把发票号显示成科学计数法
- 但如果 Excel 源文件本身就损坏了长数字精度，还是建议在源文件里先改成文本格式

## 常见问题

### 为什么切换 Sheet 后，筛选条件会被清空？

这是刻意设计的安全行为。

因为不同 Sheet 的字段含义可能完全不同，如果把上一个 Sheet 的条件继续带到下一个 Sheet，容易出现误筛。现在切换 Sheet 后会自动恢复到“不过滤”状态，避免误操作。

### 为什么预览结果里有未匹配？

常见原因有：

- Excel 里有发票号，但 PDF 文件夹里没有对应文件
- PDF 文件名格式不符合当前规则
- 同一个发票号在 PDF 中出现重复冲突
- Excel 源数据本身有空值、异常值或被错误格式化

### 为什么发票号会显示成科学计数法？

如果源 Excel 把超长发票号当数字处理，Excel 可能已经改写了显示或精度。

工具会尽量把它转回普通文本显示，但如果源文件已经丢失精度，就需要先在 Excel 里修正源数据。
