# 发票处理工具箱 v5.2.1

一个面向 Windows 的桌面发票工具，用来解决两类高频工作：

- 从复杂 Excel 工作簿中筛选需要的发票号码，再批量匹配并导出本地 PDF 发票
- 按文件名规则整理一批零散 PDF，把它们自动归档到对应公司文件夹

它适合财务、税务、出纳、行政或需要长期处理进项发票、抵扣发票、税局导出表和内部台账的人。

## 下载

## 这个工具能做什么

### 1. 发票筛选

- 打开 `.xlsx` / `.xls` 文件
- 识别一个工作簿中的多个 Sheet
- 选择目标 Sheet，提取发票列和公司列
- 在 PDF 目录中匹配同名或对应编号的发票文件
- 先预览，再正式导出
- 导出后自动生成结果报告

### 2. 多 Sheet 工作簿分析

- 展示所有 Sheet 的名称、行列规模和可用状态
- 自动识别候选发票列
- 自动识别候选公司列
- 标记推荐 Sheet
- 给出样本预览，方便确认识别结果

这部分尤其适合“一个 Excel 里混有税局表、抵扣表、内部表、月度台账”的实际场景。

### 3. 行级条件筛选

除了按发票号筛选，还可以继续按其他列缩小范围，例如：

- 只保留“是否抵扣 = 是”的发票
- 排除某些公司名称或脏数据标记
- 按某一列做“等于 / 包含 / 不等于 / 不包含”过滤

支持的模式：

- `不过滤`
- `等于任一`
- `包含任一`
- `不等于任一`
- `不包含任一`

### 4. 自动导出到当前 Excel 目录

支持按“Excel 所在目录 + 当前 Sheet 名称”自动创建导出文件夹。

例如当前 Excel 在：

```text
F:\工作文档\2025年8月
```

当前 Sheet 是：

```text
8月进项
```

那么导出目录会自动变成：

```text
F:\工作文档\2025年8月\8月进项
```

这非常适合：

- 月度进项发票筛选
- 抵扣发票筛选
- 按工作表分组归档 PDF

### 5. 发票整理

- 扫描一个文件夹里的 PDF
- 按文件名分段解析公司名称
- 自动移动到对应公司文件夹
- 支持递归扫描
- 支持预览、勾选、执行、撤销

### 6. 历史记录与回滚

- 查看整理历史
- 查看筛选历史
- 按时间、类型、关键字过滤
- 回滚导出结果
- 回滚关联报告

### 7. 桌面工作台体验

- 白天 / 黑夜双主题
- 更紧凑的默认布局
- 结果区域表格化展示
- 设置页和筛选页支持滚动
- 切换 Sheet 时自动重置条件筛选，避免误带上一个工作表的过滤条件
- v5.2.1 进一步压缩展示型头部，默认窗口下把更多空间留给实际操作区
- v5.2.0 新增更现代、统一的视觉层级与工作台风格

## 典型使用场景

### 场景一：筛选进项发票

1. 打开 Excel 文件
2. 选择 `8月进项` 这类 Sheet
3. 检查系统识别到的发票列和公司列
4. 如有需要，设置排除公司
5. 开启“按 Excel 同目录 + 当前工作表自动建导出文件夹”
6. 先预览，确认后导出

### 场景二：筛选抵扣发票

1. 打开 Excel 文件
2. 选择 `8月抵扣` 这类 Sheet
3. 条件列设为“是否抵扣”
4. 筛选模式设为“等于任一”
5. 条件值填写“是”
6. 预览结果
7. 导出匹配到的 PDF

### 场景三：批量整理零散 PDF

1. 进入“整理”
2. 选择 PDF 所在目录
3. 根据命名规则检查公司名段位
4. 扫描文件
5. 预览后执行整理
6. 如有误操作，可撤销

## PDF 命名规则

工具默认通过文件名分段来提取关键字段。

常见格式示例：

```text
dzfp_发票号码_公司名称_日期.pdf
```

默认思路：

- 使用 `_` 作为分隔符
- 在设置中配置“公司名所在段”和“发票号所在段”

如果你的 PDF 命名格式不同，也可以在设置页调整规则预设或手动修改段位。

## Excel 使用建议

工具支持复杂工作簿，但为了得到更稳定的结果，建议注意以下几点：

- 不同 Sheet 的列名可以不一致
- 自动识别不准确时，可以手动指定发票列和公司列
- 如果 Excel 已经把超长发票号按数值存储并丢失精度，任何工具都无法完全还原原始号码
- 最稳妥的做法是尽量让发票号在源表中保持文本格式

## 常见问题

### 1. 为什么发票号显示不对？

如果 Excel 源表把长号码当成数值保存，并且已经丢失精度，程序只能尽量避免科学计数法显示，不能凭空恢复被 Excel 截断的真实号码。

### 2. 为什么一个工作簿里有些 Sheet 不可用？

因为并不是每个 Sheet 都包含有效的发票列。工具会尽量识别，但如果一个 Sheet 没有合适列，就会被标记为不可筛选。

### 3. 为什么建议先预览再导出？

因为很多真实表格都混有手工标记、无效行、测试行、异常公司名。先预览可以明显减少误导出。
